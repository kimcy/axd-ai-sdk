# OpenAI × 서버 프록시

프로덕션 권장 패턴입니다. 브라우저는 자기 백엔드만 호출하고, 백엔드가 OpenAI 로 스트리밍 요청을 보낸 뒤 **axe-wire/1** 포맷으로 번역해 다시 내려줍니다.

```
Browser (5566) ──/api/chat──> Vite proxy ──> Hono (8787) ──> api.openai.com
```

- API 키가 브라우저에 노출되지 않음
- CORS 우회 (동일 출처)
- 툴 실행, 감사 로그, rate-limit, cache 를 서버에 둘 수 있음

전체 코드는 리포지토리의 [`examples/openai/`](https://github.com/kimcy/axe-ai-sdk/tree/main/examples/openai) 를 참고하세요.

## 1) 프론트 — `DefaultChatTransport`

```ts filename="src/transport.ts"
import { DefaultChatTransport } from '@axe-ai-sdk/react'

export const transport = new DefaultChatTransport({
  api: '/api/chat',
  prepareBody: (request, state) => ({
    messages: request.messages.map((m) => ({
      role: m.role,
      content: m.content,
    })),
    ...(state.conversationId ? { conversationId: state.conversationId } : {}),
    model: import.meta.env.VITE_OPENAI_MODEL ?? 'gpt-4o-mini',
  }),
})
```

`Message` 는 SDK 가 관리하는 `status`, `createdAt` 같은 UI 필드를 포함하므로 서버로 보낼 때는 `role` / `content` 만 추려서 보냅니다.

## 2) Vite 프록시

브라우저는 `/api/chat` 만 호출하고, Vite 가 백엔드로 포워딩합니다.

```ts filename="vite.config.ts"
export default defineConfig({
  plugins: [react()],
  server: {
    port: 5566,
    proxy: {
      '/api': { target: 'http://localhost:8787', changeOrigin: true },
    },
  },
})
```

## 3) 백엔드 — OpenAI → axe-wire/1

핵심은 OpenAI 스트림 청크를 [`DefaultChatTransport` 가 기대하는 이벤트](/docs/core/default-transport#sse-포맷) 로 1:1 매핑하는 것입니다.

```ts filename="server/index.ts"
import { serve } from '@hono/node-server'
import { Hono } from 'hono'
import { streamSSE } from 'hono/streaming'
import OpenAI from 'openai'
import { randomUUID } from 'node:crypto'

const openai = new OpenAI() // OPENAI_API_KEY env 자동 사용
const app = new Hono()

app.post('/api/chat', async (c) => {
  const body = await c.req.json()
  const conversationId = body.conversationId ?? randomUUID()
  const messageId = randomUUID()

  return streamSSE(c, async (stream) => {
    const ctrl = new AbortController()
    stream.onAbort(() => ctrl.abort())

    const write = (event: string, data: unknown) =>
      stream.writeSSE({ event, data: JSON.stringify(data) })

    await write('message_created', { messageId })

    try {
      const completion = await openai.chat.completions.create(
        {
          model: body.model ?? 'gpt-4o-mini',
          stream: true,
          messages: body.messages,
        },
        { signal: ctrl.signal }
      )
      for await (const chunk of completion) {
        const delta = chunk.choices[0]?.delta?.content
        if (delta) await write('message', { content: delta })
      }
      await write('done', { conversationId, conversationStatus: 'ACTIVE' })
    } catch (err) {
      const aborted = (err as Error).name === 'AbortError'
      if (!aborted) {
        await write('error', {
          error: (err as Error).message,
          code: 'OPENAI_STREAM_ERROR',
        })
      }
      await write('done', { conversationId, conversationStatus: 'ERROR' })
    }
  })
})

serve({ fetch: app.fetch, port: 8787 })
```

이벤트 흐름:

```
event: message_created
data: {"messageId":"..."}

event: message
data: {"content":"안"}

event: message
data: {"content":"녕"}

event: done
data: {"conversationId":"...","conversationStatus":"ACTIVE"}
```

`DefaultChatTransport` 가 `metadata` 에서 `conversationId` 를 자동 추적하므로, **다음 요청부터 body 에 저절로 포함**됩니다.

## 인증과 멀티 테넌시

운영에서는 로그인 쿠키로 토큰을 관리하는 경우가 흔합니다.

```ts
import { DefaultChatTransport, bearerFromCookie } from '@axe-ai-sdk/react'

new DefaultChatTransport({
  api: '/api/chat',
  headers: () => ({
    ...bearerFromCookie('auth_token')(),
    'X-Tenant': getTenantId(),
  }),
})
```

자세한 패턴은 [DefaultChatTransport — 인증](/docs/core/default-transport#인증--headers-resolver-로-토큰-주입) 을 참고하세요.

## 실행

```bash
cp examples/openai/.env.example examples/openai/.env
# OPENAI_API_KEY 를 채워넣으세요
pnpm example:openai:dev
```

- 프론트: http://localhost:5566
- API: http://localhost:8787
