# OpenAI × 브라우저 직접

백엔드 없이 **브라우저가 `api.openai.com` 에 직접 요청**하는 패턴. 로컬 데모, BYOK 플레이그라운드, 또는 인증 뒤의 내부 툴처럼 키가 노출돼도 허용되는 환경에서만 사용하세요.

```
Browser ──(openai SDK)──> api.openai.com
```

전체 코드는 [`examples/openai-browser/`](https://github.com/kimcy/axe-ai-sdk/tree/main/examples/openai-browser).

<Callout type='warning'>
  `VITE_` 접두사 환경변수는 번들에 그대로 포함됩니다. DevTools · Network 탭 ·
  소스맵에서 API 키가 보이므로 **프로덕션에서는 쓰지 마세요**. 진지한 앱이라면
  [서버 프록시 패턴](/docs/examples/openai-proxy) 을 쓰세요.
</Callout>

## `createTextStreamTransport` — 보일러플레이트를 지운다

백엔드가 없으면 `DefaultChatTransport` (SSE 와이어 파서) 는 쓸 수 없습니다. 그렇다고 `ChatTransport` 를 처음부터 구현하면 `metadata`, `message-start`, abort 분기, `finish` 종결 같은 의례적인 코드가 40줄 나옵니다.

core 의 `createTextStreamTransport` 가 그 envelope 을 대신 채워줍니다 — 사용자는 **텍스트 delta 만 yield** 하면 됩니다.

```ts filename="src/transport.ts"
import { createTextStreamTransport } from '@axe-ai-sdk/react'
import OpenAI from 'openai'

export function createOpenAITransport(opts: {
  apiKey: string
  model?: string
}) {
  const client = new OpenAI({
    apiKey: opts.apiKey,
    dangerouslyAllowBrowser: true,
  })
  const model = opts.model ?? 'gpt-4o-mini'

  return createTextStreamTransport(async function* (req) {
    const stream = await client.chat.completions.create(
      {
        model,
        stream: true,
        messages: req.messages.map((m) => ({
          role: m.role as 'system' | 'user' | 'assistant',
          content: m.content,
        })),
      },
      { signal: req.signal }
    )
    for await (const chunk of stream) {
      const delta = chunk.choices[0]?.delta?.content
      if (delta) yield delta
    }
  })
}
```

사용자가 쓴 건 **14줄**, 그중 실제 OpenAI 호출은 10줄입니다. 나머지는 core 가 처리합니다:

| core 가 자동으로 yield | 언제 |
| --- | --- |
| `metadata` (conversationId) | 최초 `send()` 또는 `reset()` 이후 첫 호출 |
| `message-start` (messageId) | 매 `send()` 시작 직후 |
| `text-delta` | 사용자가 yield 한 문자열을 래핑 |
| `finish({reason:'stop'})` | 제너레이터가 정상 종료 |
| `finish({reason:'abort'})` | `AbortError` 감지 시 |
| `error` + `finish({reason:'error'})` | 그 외 예외 |

### 옵션

```ts
createTextStreamTransport(generate, {
  conversationId: 'fixed-cid', // 고정 cid 를 쓰고 싶을 때
  conversationId: false,       // metadata 를 아예 안 내보냄 (generator 가 직접 yield)
  idFactory: () => crypto.randomUUID(), // 기본 id 생성기 교체
})
```

반환값에는 `reset()` 메서드가 붙어 있어 **대화 초기화 시 conversationId 도 갈아끼울** 수 있습니다.

### "직접 구현해야만 하는" 경우

다음 상황은 `createTextStreamTransport` 로 표현할 수 없고, `ChatTransport.send` 를 직접 구현해야 합니다.

- **툴 호출** — `tool-call` / `tool-result` / `thinking` 을 중간에 끼워넣고 싶을 때
- **인용/RAG** — `citation` StreamPart 를 발행해야 할 때
- **커스텀 메시지 id** — 서버가 내려준 messageId 를 그대로 써야 할 때

이런 경우엔 [examples/gateway/](https://github.com/kimcy/axe-ai-sdk/tree/main/examples/gateway) 의 mock-transport 를 참고하세요. `createTextStreamTransport` 는 "텍스트만 흘러가는" 단순 케이스를 위한 빠른 길입니다.

## UI

UI 코드는 [서버 프록시 예제](/docs/examples/openai-proxy) 와 **완전히 동일**합니다 — transport 만 바꿔 끼웁니다.

```tsx filename="src/App.tsx"
import { useChat, Markdown } from '@axe-ai-sdk/react'
import { createOpenAITransport } from './transport'

const transport = createOpenAITransport({
  apiKey: import.meta.env.VITE_OPENAI_API_KEY!,
})

export function App() {
  const { messages, submit, stop, isStreaming } = useChat({ transport })
  // ...
}
```

## BYOK — 사용자가 키를 직접 입력

환경변수 대신 UI 에서 키를 받아도 됩니다. 완성된 "키 게이트" 구현은 실제 예제 [`src/App.tsx`](https://github.com/kimcy/axe-ai-sdk/blob/main/examples/openai-browser/src/App.tsx) 를 보세요. 핵심은 키가 준비된 뒤에 transport 를 `useMemo` 로 생성하는 것입니다.

```tsx
const transport = useMemo(
  () => apiKey ? createOpenAITransport({ apiKey }) : null,
  [apiKey]
)
if (!transport) return <KeyGate onSubmit={setApiKey} />
return <Chat transport={transport} />
```

## 실행

```bash
pnpm example:openai-browser:dev
```

- 브라우저: http://localhost:5577
- 키는 UI 게이트에서 입력하거나, `.env.local` 에 `VITE_OPENAI_API_KEY` 를 세팅하면 스킵

Network 탭을 열어보면 **`api.openai.com` 호출이 직접 보입니다** — 서버 프록시 예제에서는 안 보이던 것입니다.
